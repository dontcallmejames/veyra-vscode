## Required Verification Commands

Run these before calling the goal complete:

```powershell
npm run verify:completion
$env:GAMBIT_RUN_LIVE = '1'
npm run test:integration:live
Remove-Item Env:\GAMBIT_RUN_LIVE -ErrorAction SilentlyContinue
```

Or run the combined all-requirements verifier:

```powershell
$env:GAMBIT_RUN_LIVE = '1'
npm run verify:goal
Remove-Item Env:\GAMBIT_RUN_LIVE -ErrorAction SilentlyContinue
```

In Bash-compatible shells, the live command is `GAMBIT_RUN_LIVE=1 npm run test:integration:live`; the combined verifier is `GAMBIT_RUN_LIVE=1 npm run verify:goal`.

`npm run verify:completion` runs the non-paid gate: `npm run verify`, `npm run test:vscode-smoke`, and `npm run verify:live-ready`. `npm run verify:goal` requires the explicit paid-prompt opt-in before non-paid checks start, then runs `npm run verify:completion` and `npm run test:integration:live`. The live-readiness step must report all entries as `ready` before running paid live tests. If any entry is `inaccessible`, fix filesystem or sandbox permissions first; it is not evidence that the CLI is missing. The same readiness and live-test commands are available in VS Code through `Gambit: Show live validation guide`. The npm live-test entrypoint first requires `GAMBIT_RUN_LIVE=1`, then runs readiness; direct `GAMBIT_RUN_LIVE=1 vitest ...` invocations are also guarded by `tests/integration/liveReadinessGuard.ts`, which throws in `beforeAll` before any prompt body runs when readiness is incomplete. In PowerShell, `$env:GAMBIT_RUN_LIVE = '1'` stays set for the current terminal session until you remove it or close the shell.
